Face Swap AI

  In Constrained Environment

Face Swap Idea

The idea was to create a Face swap AI that can be deployed in Constrained Environments. It swaps the face of the input video with the input image.

Input-Data

Input Video:

We split the video into frames
and then crop those frames with
the help of a face align algo.
The neural network only works
proper when the face is in the
middle of the screen. After
we found the best position
and cropped it we put the
cropped frames back together.

Face Swap NN:

Now we can put our prepared video and our cropped image into the neural network, which changes specific attributes that can be seen on the colormap. You can swap the whole face and also just the lips, hair a.s.o.

Usage:

First, you have to process your video so
the face is in the middle. Then you have
to align your imported image. At last, you
can then start the prediction, which will
be automatically saved.

Optimisations:

The code has already been optimized.
The Picture and Video should be taken
in good lighting for a good result.
Too much head movement and a
wobbly Video will worsen the result
immensely.

Output Examples

Deployment of a NN on an ESP:

We deploy, understand and modify the ESP-Who Web server example. We tried to optimize the performance when the model detects or recognizes a face. As well as setting names for the Subjects.

The camera takes a picture in full resolution, then crops it down under 500x500 pixels when the AI features are active. This uses up a lot of resources. Instead, we can set the sensor and monitor size to a more efficient resolution when we want the AI functionalities. This gets us a noticeable better performance.

We hacked together the C, HTML and Query String code to set a name to every subject. Due to the size of the video, the maximum length of a name is 11 Characters.
